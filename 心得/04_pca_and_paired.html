04 / PCA & Paired

paired color-gray 分析把“同一张图的两种版本”显式用起来

`testcode/task1_paired_decoding` 的价值，在于它不再把 color 和 gray 只当作两组独立 trial，而是把同一张图片的彩色版与灰度版配成一对。这样既能做更针对性的二分类，也能做更干净的 PCA 解释和相似度追踪。

groupedData 前提

配对分析成立的前提，是 pair id 与保留 trial 对齐

`groupedData` 约定为 4 × 2 cell：行对应类别，列对应 color / gray。同一行里同一个数值 id 表示同一张图片。

理想输入

如果 trial 没有被剔除，那么每个 cell 内部的位置就直接等于 repeat index。但当前主线会先做 bad epoch 剔除和 min-trial 截断，所以这个假设通常不再自动成立。

当前实现

paired 脚本会先读取 `processed_data` 中保存的 repeat 元数据，再把 `groupedData` 重排到 ROI 张量真正保存下来的 repeat 轴，然后才开始匹配 pair。

PCA 原理图

PCA 在这里不是降维炫技，而是为了让颜色效应更可视

PCA 在这些脚本里的职责很清楚：把高维时间窗或均值向量压缩到 2D/3D，以便观察 color 与 gray 是否沿着同一主轴分离、不同类别是否走出不同轨迹。

三种 paired decoding

不同方案实际上对应不同的研究假设

脚本 核心操作 最适合回答的问题 主要风险
paired_group_cv_decoding.py 使用原始 color / gray trial，但在分折时保持同一 pair 进同一 fold 如果不去掉图像内容，只要求“同图不同颜色”能否被分开 图像本身的共享结构仍然存在，颜色效应可能被图像主效应覆盖
within_pair_centered_decoding.py 每对样本都减去 pair mean，再做 color-gray 分类 去掉图像共享部分之后，残差信号是否还能区分颜色状态 如果 pair 本身噪声很高，中心化会同时放大不稳定性
pair_difference_decoding.py 构造 `+(color-gray)` 与 `-(color-gray)` 两类 signed difference 最直接地抓颜色差异向量，问颜色效应是否本身可解码 最依赖 pair 对齐质量，对错配和单对异常值都更敏感

怎么读这些图

PCA 和余弦图背后的最小解释框架

如果 raw PCA 已经明显分离

说明颜色状态确实在总体信号空间里占据一定方差方向，但这时还不能说明它脱离了图像内容。

如果 centered PCA 仍然分离

说明扣掉 pair shared content 之后，颜色效应仍有稳定几何结构，这比 raw 分离更有解释力。

如果 cosine similarity 下降

说明 color 与 gray 在该类别上的空间方向开始分叉。若 centered similarity 也下降，更支持颜色本身在改变表征。

如果 trajectory 轨迹分开但 decoding 不显著

说明均值结构存在可视差异，但 trial-level 波动或样本量可能还不足以支撑稳定分类。

paired analysis 的关键不是“让分类更高”，而是让颜色效应与图像内容尽量分开解释。PCA、trajectory 和 cosine similarity 都是在帮你回答同一个问题：颜色改变的是平均位置、变化方向，还是两者都有。
